# Designable 整体架构深度解析

## 一、项目概述

Designable 是一个由阿里巴巴开源的**低代码设计器引擎**，专门用于构建可视化的表单设计器和页面搭建工具。它提供了完整的拖拽、编辑、预览能力，并深度集成了 Formily 表单解决方案。

### 1.1 核心特性

- **完整的设计器引擎**：提供拖拽、选择、历史记录、快捷键等核心能力
- **响应式架构**：基于 @formily/reactive 实现的响应式状态管理
- **多框架支持**：支持 Ant Design 和 Alibaba Next 组件库
- **沙箱隔离**：提供 iframe 沙箱用于安全预览
- **高度可扩展**：通过 Registry 系统支持自定义行为、图标、语言等
- **Monorepo 架构**：使用 Lerna + Yarn Workspaces 管理多包

### 1.2 技术栈

- **语言**：TypeScript 4.1.5
- **框架**：React 16.x || 17.x
- **状态管理**：@formily/reactive（响应式系统）
- **UI 组件库**：Ant Design 4.0.0+ / Alibaba Next 1.23.0+
- **编辑器**：Monaco Editor 0.25.2+
- **构建工具**：TypeScript Compiler + Rollup + Webpack

## 三、核心设计模式

### 3.1 观察者模式 (Observer Pattern)

Designable 大量使用观察者模式来实现事件驱动和状态响应。

#### 3.1.1 Subscribable 订阅系统

位置：`packages/shared/src/subscribable.ts`

```typescript
export class Subscribable<ExtendsType = any> {
  private subscribers: {
    index?: number
    [key: number]: ISubscriber
  } = { index: 0 }

  // 分发事件给所有订阅者
  dispatch<T extends ExtendsType = any>(event: T, context?: any) {
    let interrupted = false
    for (const key in this.subscribers) {
      if (isFn(this.subscribers[key])) {
        event['context'] = context
        if (this.subscribers[key](event) === false) {
          interrupted = true
        }
      }
    }
    return interrupted ? false : true
  }

  // 订阅事件
  subscribe(subscriber: ISubscriber) {
    let id: number
    if (isFn(subscriber)) {
      id = this.subscribers.index + 1
      this.subscribers[id] = subscriber
      this.subscribers.index++
    }
    return () => this.unsubscribe(id)
  }
}
```

**设计亮点**：
- 使用数字索引管理订阅者，避免数组操作的性能问题
- 返回取消订阅函数，符合 React Hooks 的使用习惯
- 支持中断机制（返回 false 可中断后续订阅者）

#### 3.1.2 响应式系统

基于 `@formily/reactive` 实现的响应式状态管理：

```typescript
// TreeNode 使用 observable 装饰器
makeObservable() {
  define(this, {
    componentName: observable.ref,
    props: observable,
    hidden: observable.ref,
    children: observable.shallow,
    designerProps: observable.computed,
    designerLocales: observable.computed,
    wrap: action,
    prepend: action,
    append: action,
    // ...
  })
}
```

**设计亮点**：
- `observable.ref`：引用类型的响应式，适合不可变数据
- `observable.shallow`：浅层响应式，适合数组
- `observable.computed`：计算属性，自动缓存和依赖追踪
- `action`：标记修改操作，批量更新优化

### 3.5 命令模式 (Command Pattern)

History 系统实现了撤销/重做功能。

位置：`packages/core/src/models/History.ts`

```typescript
export class History<T = any> {
  list: IHistoryItem<T>[] = []
  current = 0

  push(type?: string) {
    // 保存当前状态
    const item = this.serialize(type)
    this.list = this.list.slice(0, this.current + 1)
    this.list.push(item)
    this.current++
  }

  undo() {
    if (this.allowUndo) {
      this.current--
      this.goTo(this.current)
    }
  }

  redo() {
    if (this.allowRedo) {
      this.current++
      this.goTo(this.current)
    }
  }
}
```

**设计亮点**：
- 快照式存储，每次操作保存完整状态
- 使用 requestIdleCallback 优化性能
- 支持历史记录锁定，避免循环触发

## 四、核心模型详解

### 4.1 Engine（引擎）

Engine 是整个设计器的核心控制器，负责：

- **生命周期管理**：mount/unmount
- **事件分发**：统一的事件处理入口
- **全局状态**：cursor、keyboard、screen
- **工作台管理**：workbench

```typescript
export class Engine extends Event {
  id: string
  props: IEngineProps<Engine>
  cursor: Cursor          // 光标状态
  workbench: Workbench    // 工作台（包含多个工作区）
  keyboard: Keyboard      // 键盘状态
  screen: Screen          // 屏幕状态（PC/Mobile/Tablet）

  mount() {
    this.attachEvents(globalThisPolyfill)
  }

  unmount() {
    this.detachEvents()
  }
}
```

### 4.3 Workspace（工作区）

Workspace 是单个设计器实例，包含：

- **viewport**：主视口（画布区域）
- **outline**：大纲视口（树形结构）
- **operation**：操作对象（包含 tree、selection、hover 等）
- **history**：历史记录

TreeNode 是设计树的基本单元：

```typescript
export class TreeNode {
```

## 五、事件驱动系统

### 5.1 事件流

```
用户交互（鼠标/键盘）
    ↓
EventDriver（事件驱动器）
    ↓
CustomEvent（自定义事件）
    ↓
Effect（副作用处理器）
    ↓
Model 状态更新
    ↓
React 组件重渲染
```

### 5.2 Driver 系统

Driver 负责监听原生事件并转换为自定义事件：

- **DragDropDriver**：拖拽事件（DragStartEvent、DragMoveEvent、DragStopEvent）
- **MouseClickDriver**：鼠标点击事件
- **MouseMoveDriver**：鼠标移动事件
- **KeyboardDriver**：键盘事件
- **ViewportResizeDriver**：视口大小变化
- **ViewportScrollDriver**：视口滚动

### 5.3 Effect 系统

Effect 处理自定义事件并更新模型状态：

- **useDragDropEffect**：处理拖拽逻辑
- **useSelectionEffect**：处理选择逻辑
- **useResizeEffect**：处理调整大小
- **useTranslateEffect**：处理平移
- **useContentEditableEffect**：处理内容编辑

## 八、性能优化策略

### 8.1 响应式优化

- **observable.shallow**：数组使用浅层响应式，避免深度遍历
- **observable.computed**：计算属性自动缓存
- **action 批量更新**：多个状态修改合并为一次更新

### 8.2 事件优化

- **requestIdleCallback**：历史记录快照使用空闲时间
- **事件节流**：鼠标移动事件去重
- **批量事件监听**：batchAddEventListener 减少重复绑定

### 8.3 渲染优化

- **虚拟滚动**：大纲树使用虚拟滚动
- **按需渲染**：只渲染可见区域
- **memo 优化**：React 组件使用 memo 避免不必要的重渲染

## 十、总结

### 10.1 架构优势

1. **分层清晰**：核心引擎、UI 层、集成层职责明确
2. **高度解耦**：通过事件驱动实现模块间通信
3. **响应式**：基于 @formily/reactive 的响应式系统
4. **可扩展**：Registry 系统支持灵活扩展
5. **类型安全**：完整的 TypeScript 类型定义

### 10.2 设计模式总结

- **观察者模式**：事件系统、响应式系统
- **组合模式**：TreeNode 树结构
- **策略模式**：Driver 系统
- **注册表模式**：GlobalRegistry
- **命令模式**：History 系统
- **工厂模式**：Engine 创建模型

### 10.3 适用场景

- **表单设计器**：可视化表单搭建
- **页面搭建器**：低代码页面编辑
- **配置化平台**：可视化配置工具
- **原型设计工具**：快速原型搭建
